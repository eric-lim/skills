# Severity Calibration Guide

Categorize every finding by its actual impact using these definitions:

- **Critical:** Bugs, logical/correctness errors, security vulnerabilities, or data loss risks that will break production, cause incorrect execution, or prevent compilation.
- **Important:** Architectural flaws, missing requirements, poor error handling, test gaps, or major deviations from standard practices.
- **Minor:** Code style issues, optimization opportunities, pedantic nitpicks, formatting issues, or minor documentation polish.
